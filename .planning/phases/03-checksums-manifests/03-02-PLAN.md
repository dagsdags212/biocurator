<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| manifest.json→verifier | Manifest file is user-controlled input read from disk — may be malformed, malicious, or from a different job |
| manifest paths→filesystem | Manifest file entries specify relative paths that are resolved against the manifest directory — path traversal risk |
| disk file→verifier | Files opened read-only for SHA-256 re-computation; concurrent writes by other processes may produce hash mismatch |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05 | Tampering | manifest_verify() — path traversal in file entries | mitigate | Reject paths containing `..` or starting with `/` before resolution. Log warning and skip entry. Per RESEARCH.md V5 input validation. |
| T-03-06 | Denial of Service | manifest_verify() — oversized manifest JSON | mitigate | Use `json.loads()` (not streaming parser); for a single-user CLI tool with typical <1000 file entries, this is not a practical DoS vector. Documented as accepted risk for v1. |
| T-03-07 | Tampering | manifest_verify() — malicious manifest with crafted file paths pointing to sensitive files | mitigate | Same traversal gate as T-03-05 catches `../` patterns. Absolute paths (`/etc/passwd`) also rejected. Only sibling-or-descendant files accessible. |
| T-03-08 | Spoofing | manifest_verify() — SHA-256 collision attack replacing files | accept | SHA-256 has 2^128 collision resistance. Computationally infeasible to generate a collision for arbitrary biological sequence files. Not a practical threat. |
| T-03-09 | Denial of Service | manifest_verify() — file read blocks on large files | accept | Single-user CLI tool runs locally. File read is bounded by local I/O speed; no network amplification possible. |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No new dependencies — hashlib, json, pathlib are all Python stdlib. |
</threat_model>
