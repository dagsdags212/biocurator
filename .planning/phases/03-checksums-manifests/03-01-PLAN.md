## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| exporter→filesystem | Manifest JSON written to user-specified output directory; manifest paths resolved relative to outdir |
| curator→exporter | JobConfig passed as dataclass — serialized via asdict() for manifest embedding |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | StreamingExporter._write_manifest() | mitigate | Manifest write failure is caught via try/except OSError, logged as warning; data files already on disk are unaffected — missing manifest is recoverable |
| T-03-02 | Tampering | Manifest file modification after write | accept | Manifest is a provenance record, not a security control. Phase 4 `manifest_verify()` detects modification by recomputing checksums from disk. No OS-level file locking needed for single-user CLI tool. |
| T-03-03 | Information Disclosure | Config snapshot in manifest JSON | accept | JobConfig does not contain secrets (no API keys, no passwords). Email is a public identifier per NCBI guidelines. Config snapshot is for provenance — no PII exposure risk. |
| T-03-04 | Denial of Service | Large manifest JSON | accept | Single-user CLI tool; manifest size is proportional to job output (usually <1000 files). No external input creates unbounded manifest growth. |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No new dependencies — hashlib, json, dataclasses, datetime, io, pathlib are all Python stdlib. |
</threat_model>
