**CRITICAL — correct mock targets for inline imports:**

`health.py` uses **inline imports** inside method bodies (NOT module-level imports):
- `ping_ncbi()` line 19: `from Bio import Entrez` — executed at call time, resolves `Bio.Entrez` through `sys.modules`
- `ping_uniprot()` line 44: `import requests` — executed at call time, resolves `requests` through `sys.modules`

Therefore, `unittest.mock.patch` on module-level attributes (`biocurator.providers.health.Entrez`, `biocurator.providers.health.requests`) does **NOT** intercept these inline imports — Python resolves inline imports through `sys.modules`, bypassing module-attribute patches. The correct strategies:

**For NCBI tests (ping_ncbi):** Patch `Bio.Entrez.esearch` and `Bio.Entrez.read` directly. Since `from Bio import Entrez` resolves `Bio.Entrez` from `sys.modules`, patching attributes on `Bio.Entrez` works because `Entrez` inside the function is a reference to the same module object.

```python
@patch("Bio.Entrez.read")
@patch("Bio.Entrez.esearch")
def test_ping_ncbi_reachable(mock_esearch, mock_read):
    mock_handle = MagicMock()
    mock_esearch.return_value = mock_handle
    mock_read.return_value = {"IdList": ["123"]}
    result = HealthChecker.ping_ncbi(timeout=30)
    assert result.provider == "ncbi"
    assert result.reachable is True
    assert result.response_time_ms > 0
    assert result.error is None
```

**For UniProt tests (ping_uniprot):** Use `patch.dict('sys.modules', {'requests': mock_requests})` to replace the `requests` module in `sys.modules` before the function call. Since `import requests` resolves through `sys.modules`, this intercepts the inline import.

```python
@patch.dict("sys.modules", {})
def test_ping_uniprot_reachable(mock_modules):
    mock_requests = MagicMock()
    mock_response = MagicMock()
    mock_response.raise_for_status.return_value = None  # no exception = success
    mock_requests.get.return_value = mock_response
    mock_modules["requests"] = mock_requests
    
    result = HealthChecker.ping_uniprot(timeout=30)
    assert result.provider == "uniprot"
    assert result.reachable is True
    assert result.response_time_ms > 0
    assert result.error is None

@patch.dict("sys.modules", {})
def test_ping_uniprot_unreachable(mock_modules):
    mock_requests = MagicMock()
    mock_requests.get.side_effect = Exception("Timeout")
    mock_modules["requests"] = mock_requests
    
    result = HealthChecker.ping_uniprot(timeout=30)
    assert result.provider == "uniprot"
    assert result.reachable is False
    assert result.response_time_ms > 0
    assert result.error == "Timeout"
```

Note: `patch.dict("sys.modules", {})` replaces all sys.modules entries for the duration of the test. This is intentional — it clears cached modules so `import requests` inside `ping_uniprot()` picks up our mock. Only the `requests` key is set; `Bio.Entrez` remains unset in sys.modules, but NCBI tests use `@patch("Bio.Entrez.esearch")` which adds the mock to the real `Bio.Entrez` module object, so no conflict.
