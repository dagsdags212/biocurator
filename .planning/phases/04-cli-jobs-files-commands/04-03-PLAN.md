# Plan 04-03: `biocurator files --verify` Mode

## Objective

Extend `files_command` with checksum verification via the existing `manifest_verify()` library function. When `--verify` is passed, re-read all relevant files from disk, recompute SHA-256 checksums, compare against manifest entries, and display a per-file status table. Exit code 1 if any file is corrupted or missing. Add corresponding tests to `tests/cli/test_files.py`.

## Background

`manifest_verify()` is already implemented in `src/biocurator/core/verifier.py` and exported via `src/biocurator/core/__init__.py`. It accepts a `Path` to `manifest.json` and returns:
```python
{
    "manifest_valid": bool,
    "files_checked": int,
    "files_matched": int,
    "files_missing": int,
    "files_corrupted": int,
    "all_ok": bool,
    "results": [{"path": str, "status": "ok"|"corrupted"|"missing", "sha256_expected": str, "sha256_actual": str|None}]
}
```
Plan 04-02 created `files_command` with a stub `if verify: ... return` branch. This plan replaces that stub with the real implementation.

## must_haves

```yaml
truths:
  - "biocurator files --verify --config {cfg}" with all checksums matching exits 0 and output contains "All checksums verified"
  - "biocurator files --verify --config {cfg}" with a corrupted file exits 1 and output contains "corrupted"
  - "biocurator files --verify --config {cfg}" with a missing file exits 1 and output contains "missing"
  - "biocurator files job-name --verify --config {cfg}" verifies only that job's manifest
  - pytest tests/cli/test_files.py -q exits 0 with all verify-mode tests passing
```

**When `job_name` is provided + `verify=True`:**
- Find matching job (same error as list mode if not found)
- `manifest_path = Path(job.export.outdir) / "manifest.json"`
- If manifest_path doesn't exist: `console.print(f"[yellow]No manifest for '{job_name}' — run the job first.[/yellow]")` then `return`
- Call `result = manifest_verify(manifest_path)`
- If `not result["manifest_valid"]`: `console.print(f"[bold red]Cannot parse manifest:[/bold red] {manifest_path}")` then `raise typer.Exit(1)`
- Build verify table: `title=f"Verification — {job_name}"`, `show_header=True`, `header_style="bold magenta"`
  Columns: `"Filename"` (style="cyan"), `"Status"`, `"SHA-256 Match"`, `"Notes"`
  For each entry in `result["results"]`:
    - status `"ok"` → status_display=`"[bold green]✓ ok[/bold green]"`, match=`"[bold green]✓[/bold green]"`, notes=`""`
    - status `"corrupted"` → status_display=`"[bold red]✗ corrupted[/bold red]"`, match=`"[bold red]✗[/bold red]"`, notes=`f"expected: {entry['sha256_expected'][:12]}..."`
    - status `"missing"` → status_display=`"[bold yellow]? missing[/bold yellow]"`, match=`"[dim]N/A[/dim]"`, notes=`"File not found on disk"`
  Add row: `entry["path"]`, status_display, match, notes
- Print table, then:
  - If `result["all_ok"]`: `console.print("[bold green]All checksums verified ✓[/bold green]")`
  - Else: `console.print(f"[bold red]Issues found: {result['files_corrupted']} corrupted, {result['files_missing']} missing[/bold red]")` then `raise typer.Exit(1)`

**When `job_name` is None + `verify=True`:**
- For each job in `global_config.jobs` where `(Path(job.export.outdir) / "manifest.json").exists()`:
  - Call `manifest_verify(manifest_path)` for each
  - Collect all results
- If no manifests found: `console.print("[yellow]No manifests found — run jobs first.[/yellow]")` then `return`
- Show combined verify table with an additional `"Job"` column prepended
- Footer: all_ok across all jobs → green success; any failure → red failure + `raise typer.Exit(1)`
</action>

Helper function to add to the test file (after `_write_manifest`):
```python
def _write_manifest_with_real_files(outdir: Path, job_name: str) -> None:
    """Write manifest.json with real matching files for verify-ok tests."""
    outdir.mkdir(parents=True, exist_ok=True)
    content = b">seq1\nATGC\n"
    fasta = outdir / "alpha_sequences.fasta"
    fasta.write_bytes(content)
    import hashlib
    sha256 = hashlib.sha256(content).hexdigest()
    manifest = {
        "manifest_version": "1.0",
        "job_name": job_name,
        "generated_at": "2026-01-01T00:00:00+00:00",
        "config": {},
        "databases": ["ncbi"],
        "stats": {"total_records": 1, "total_files": 1},
        "files": [{"path": "alpha_sequences.fasta", "format": "fasta", "sha256": sha256, "size": len(content), "record_count": 1, "provider": ["ncbi"]}],
    }
    (outdir / "manifest.json").write_text(json.dumps(manifest))
```

Four test functions:

1. `test_files_verify_ok(tmp_path)`:
   - Write config with `alpha_dir = str(tmp_path / "alpha")`
   - Call `_write_manifest_with_real_files(tmp_path / "alpha", "alpha-job")`
   - Invoke `["files", "alpha-job", "--verify", "--config", str(cfg)]`
   - Assert `result.exit_code == 0` and `"All checksums verified"` in `result.output`

2. `test_files_verify_corrupted_exits_1(tmp_path)`:
   - Write config, write `_write_manifest(tmp_path / "alpha", "alpha-job")` (wrong checksum `"a"*64`)
   - Write actual file: `(tmp_path / "alpha" / "alpha_sequences.fasta").write_bytes(b">seq1\nATGC\n")`
   - Invoke `["files", "alpha-job", "--verify", "--config", str(cfg)]`
   - Assert `result.exit_code == 1` and `"corrupted"` in `result.output.lower()`

3. `test_files_verify_missing_exits_1(tmp_path)`:
   - Write config, write manifest (real checksum but no actual file on disk)
   - `(tmp_path / "alpha").mkdir()`; write manifest referencing `"alpha_sequences.fasta"` but don't create the file
   - Invoke `["files", "alpha-job", "--verify", "--config", str(cfg)]`
   - Assert `result.exit_code == 1` and `"missing"` in `result.output.lower()`

4. `test_files_verify_no_manifest_shows_hint(tmp_path)`:
   - Write config, empty outdir (no manifest)
   - Invoke `["files", "alpha-job", "--verify", "--config", str(cfg)]`
   - Assert `result.exit_code == 0` (not an error — just no data yet) and `"run"` in `result.output.lower()`
</action>
