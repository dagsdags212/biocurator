# Plan 04-01: `biocurator jobs` Command

## Objective

Implement `biocurator jobs [--config PATH]` — list all curation jobs defined in a YAML config, defaulting to `biocurator_config.yaml` in CWD. Render a Rich table with job metadata. Exit code 1 with a clear `--config` hint if the default config is not found.

## Background

Phase 3 is complete. The config schema (`GlobalConfig.jobs: list[JobConfig]`) and `ConfigLoader` are stable. The `status` command (`src/biocurator/cli/commands/status.py`) is the canonical analog: same config-load pattern, same Rich table style, same error handling. `main.py` registers commands with `app.command("name")(fn)`.

## must_haves

```yaml
truths:
  - "biocurator jobs --config {path}" exits 0 and displays a Rich table containing every job name defined in the config
  - "biocurator jobs" with no args when "biocurator_config.yaml" does not exist exits non-zero and output contains "--config"
  - jobs_command is registered as app.command("jobs") in src/biocurator/cli/main.py
  - pytest tests/cli/test_jobs.py -q exits 0 with ≥4 passing tests
```
